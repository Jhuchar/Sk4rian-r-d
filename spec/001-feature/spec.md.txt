# Spec - Autoorientación y Normalización de Escaneos OMR

## Resumen ejecutivo (≤150 palabras)

Se propone incorporar una funcionalidad de autoorientación y normalización de hojas ópticas antes del procesamiento OMR.

Actualmente, las fichas pueden ser escaneadas con rotaciones o inclinaciones debido a errores operativos de los tutores o limitaciones de los equipos de digitalización. Esto provoca descartes innecesarios y reprocesamientos manuales.

La funcionalidad deberá detectar automáticamente los puntos de referencia de cada hoja, corregir su inclinación y orientación, y continuar con el procesamiento cuando exista evidencia suficiente para determinar la posición correcta.

Las correcciones realizadas deberán registrarse para fines de auditoría interna. Las páginas que no contengan puntos de referencia válidos serán descartadas indicando la causa correspondiente.

La funcionalidad se considera exitosa cuando todas las páginas con información suficiente son corregidas y procesadas correctamente, aun cuando otras páginas deban ser descartadas por problemas de escaneo.

---

# 1. Contexto de negocio (qué y por qué)

## Problema que resuelve

Durante el proceso de digitalización, los tutores pueden escanear hojas ópticas con diferentes orientaciones o inclinaciones.

Entre los problemas más frecuentes se encuentran:

* Rotaciones de 90°, 180° o 270°.
* Hojas inclinadas.
* Desalineaciones producidas por el escáner.
* Mezcla de orientaciones dentro de un mismo PDF.

Actualmente estos casos pueden provocar descartes innecesarios o requerir nuevos escaneos.

## Por qué ahora

El volumen de procesamiento aumenta continuamente y los errores de escaneo generan reprocesamientos que consumen tiempo operativo.

La corrección automática permitirá reducir la dependencia del proceso manual y aumentar la cantidad de fichas procesadas correctamente.

## A quién impacta

* Asistentes de sistemas.
* Tutores responsables del escaneo.
* Personal de validación.
* Estudiantes cuyos resultados dependen del procesamiento correcto de las fichas.

---

# 2. User Stories y Criterios de Aceptación

## US-1 (P1)

Como asistente de sistemas, quiero que las hojas ópticas sean orientadas automáticamente antes de su procesamiento para reducir errores provocados durante el escaneo.

### AC-1.1

Dado una hoja rotada 90°, cuando el sistema detecte sus puntos de referencia, entonces deberá corregir automáticamente la orientación y procesar la hoja.

### AC-1.2

Dado una hoja rotada 180°, cuando el sistema detecte sus puntos de referencia, entonces deberá corregir automáticamente la orientación y procesar la hoja.

### AC-1.3

Dado una hoja rotada 270°, cuando el sistema detecte sus puntos de referencia, entonces deberá corregir automáticamente la orientación y procesar la hoja.

### AC-1.4

Dado una hoja inclinada, cuando el sistema detecte los puntos de referencia necesarios, entonces deberá corregir la inclinación antes del procesamiento.

---

## US-2 (P1)

Como asistente de sistemas, quiero que cada página sea evaluada de forma independiente para evitar que errores aislados afecten al resto del documento.

### AC-2.1

Dado un PDF con páginas válidas e inválidas, cuando se procese el documento, entonces cada página deberá evaluarse individualmente.

### AC-2.2

Dado un PDF donde algunas páginas no puedan procesarse, cuando finalice el procesamiento, entonces las páginas válidas deberán conservar sus resultados.

---

## US-3 (P1)

Como asistente de sistemas, quiero conocer qué correcciones automáticas fueron aplicadas para fines de auditoría y mejora operativa.

### AC-3.1

Dado una página cuya orientación fue corregida, cuando finalice el procesamiento, entonces deberá registrarse que se aplicó una corrección automática.

### AC-3.2

Dado un procesamiento completado, cuando se consulte el detalle del lote, entonces deberá poder identificarse qué páginas requirieron corrección de orientación.

---

## US-4 (P1)

Como tutor o asistente de sistemas, quiero identificar las páginas descartadas para poder realizar acciones correctivas posteriores.

### AC-4.1

Dado una página sin puntos de referencia válidos, cuando se procese el documento, entonces la página deberá descartarse.

### AC-4.2

Dado una página descartada, cuando se consulte el resultado del procesamiento, entonces deberá mostrarse el motivo del descarte.

---

# 3. Requisitos No Funcionales (NFR)

### NFR-1

La corrección automática no deberá alterar las marcaciones originales de la hoja.

### NFR-2

Toda corrección automática aplicada deberá quedar registrada para auditoría.

### NFR-3

La evaluación de una página no deberá afectar el resultado de otras páginas pertenecientes al mismo PDF.

### NFR-4

El sistema no deberá inferir orientaciones cuando no exista evidencia suficiente para determinar la posición correcta.

### NFR-5

La funcionalidad deberá respetar los principios definidos en la Constitución de System OMR.

---

# 4. Casos Borde

* PDF que contiene páginas con diferentes orientaciones.
* Página rotada 180°.
* Página rotada 90°.
* Página rotada 270°.
* Página con inclinación significativa.
* Página sin puntos de referencia.
* Página en blanco.
* PDF con mezcla de páginas válidas e inválidas.
* PDF donde todas las páginas sean descartadas.
* PDF donde todas las páginas requieran corrección automática.

---

# 5. Assumptions

* Se asume que las hojas válidas contienen puntos de referencia detectables.
* Se asume que la barra de orientación permite determinar la posición correcta de la hoja.
* Se asume que la orientación puede corregirse utilizando la geometría obtenida a partir de los puntos de referencia.
* Si los puntos de referencia no son detectables, la página será descartada.
* Si una página es descartada, ello no afectará el procesamiento de las demás páginas del documento.
